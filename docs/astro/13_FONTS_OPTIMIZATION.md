# Font Optimization

## Current State

- Google Fonts loaded externally via `<link>` tags (`Noto Sans JP`, `M PLUS Rounded 1c`)
- CSP allows `https://fonts.googleapis.com` / `https://fonts.gstatic.com`
- CSS variables define font stacks:
  - `--font-sans: "Noto Sans JP", "Hiragino Kaku Gothic Pro", system-ui, sans-serif`
  - `--font-heading: "M PLUS Rounded 1c", "Noto Sans JP", sans-serif`

## Issues

1. **External requests**: Round trips to Google Fonts CDN block rendering
2. **CSP relaxation**: Must whitelist `fonts.googleapis.com` / `fonts.gstatic.com`
3. **FOUT/FOIT**: Layout shifts during font loading
4. **No selective loading**: No mechanism to load only required weights

## Improvement: Astro 6 `fonts` Configuration

Use Astro 6's built-in font management. Font files are downloaded at build time and self-hosted.

### Configuration Example

```typescript
// astro.config.ts
import { defineConfig, fontProviders } from "astro/config";

export default defineConfig({
  fonts: [
    {
      provider: fontProviders.google(),
      name: "Noto Sans JP",
      cssVariable: "--font-sans",
      weights: ["400", "500", "700"],
      fallbacks: ["Hiragino Kaku Gothic Pro", "system-ui", "sans-serif"],
    },
    {
      provider: fontProviders.google(),
      name: "M PLUS Rounded 1c",
      cssVariable: "--font-heading",
      weights: ["700", "800"],
      fallbacks: ["Noto Sans JP", "sans-serif"],
    },
  ],
});
```

### Benefits

| Item | Before | After |
|------|--------|-------|
| Font delivery | Google CDN (external) | Self-hosted (_astro/) |
| CSP | Must allow `fonts.googleapis.com` | `'self'` only |
| CLS | FOUT/FOIT present | Auto-mitigated with `optimizedFallbacks` |
| Requests | Round trips to separate domain | Same origin |

### Optimized Fallbacks

Enabled by default. Astro analyzes font metrics and automatically adjusts local fallback sizes to minimize CLS.

```typescript
{
  name: "Noto Sans JP",
  cssVariable: "--font-sans",
  optimizedFallbacks: true, // Default: true
}
```

### Local Font Usage

Local font files can be managed with the same API:

```typescript
{
  provider: fontProviders.local(),
  name: "CustomFont",
  cssVariable: "--font-custom",
  options: {
    variants: [{
      src: ["./src/assets/fonts/CustomFont.woff2"],
      weight: "normal",
      style: "normal",
    }],
  },
}
```

### Available Providers

| Provider | Purpose |
|----------|---------|
| `fontProviders.google()` | Google Fonts |
| `fontProviders.fontsource()` | Fontsource (OSS fonts) |
| `fontProviders.adobe()` | Adobe Fonts |
| `fontProviders.bunny()` | Bunny Fonts (GDPR compliant) |
| `fontProviders.local()` | Local font files |

### `<Font />` Component in Layout

After configuring fonts in `astro.config.ts`, add the `<Font />` component to the page `<head>` to activate them:

```astro
---
// layouts/Base.astro
import { Font } from "astro:assets";
---
<html>
  <head>
    <Font cssVariable="--font-sans" />
    <Font cssVariable="--font-heading" />
    <!-- other head elements -->
  </head>
  <body>
    <slot />
  </body>
</html>
```

This generates optimized `<link rel="preload">` tags and `@font-face` declarations automatically.

### Programmatic Font Data Access

The `fontData` export from `astro:assets` provides low-level access to font metadata. Useful for generating OG images with libraries like `satori`:

```typescript
// pages/api/og.png.ts
import type { APIRoute } from "astro";
import { fontData } from "astro:assets";
import satori from "satori";

export const GET: APIRoute = async (context) => {
  const data = fontData["--font-sans"];
  const svg = await satori(/* ... */, {
    fonts: [{
      name: "Noto Sans JP",
      data: await fetch(
        new URL(data[0].src[0].url, context.url.origin),
      ).then((res) => res.arrayBuffer()),
      weight: 400,
      style: "normal",
    }],
  });
  // ... convert SVG to PNG
};
```

## CSS Variable Changes

Font variable definitions in `app.css` become unnecessary as Astro `fonts` configuration auto-generates them:

```css
/* Before: Manual definition */
:root {
  --font-sans: "Noto Sans JP", "Hiragino Kaku Gothic Pro", system-ui, sans-serif;
  --font-heading: "M PLUS Rounded 1c", "Noto Sans JP", sans-serif;
}

/* After: Auto-generated by Astro fonts — remove manual definitions */
```

Tailwind CSS v4 font settings can continue referencing `--font-sans` / `--font-heading` as-is.

## CSP Header Updates

Since fonts are self-hosted, remove external domains from CSP:

```typescript
// Before
"style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com"

// After
"style-src 'self' 'unsafe-inline'; font-src 'self'"
```
